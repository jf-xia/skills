# 视觉驱动与性能

## 结构化感知优先级

1. `GET /source` — 默认首选，完整页面树
2. `GET /wda/accessibleSource` — 页面树过大时，精简可访问元素
3. `GET /screenshot` — 无障碍信息不足、坐标需视觉判断时

```bash
# 带过滤参数
curl "http://localhost:8100/session/$SESSION_ID/source?format=xml&excluded_attributes=frame"
```

## 视觉闭环

1. **Observe**：截图或拉取页面树
2. **Plan**：根据文本/结构/视觉确定目标
3. **Act**：调用点击、输入、滚动等接口
4. **Check**：再次截图或读取 source 确认生效

结构化信息不足时，此闭环比单次截图+坐标点击更稳。

## 策略选择

| 情况 | 策略 |
|------|------|
| 页面树稳定、节点可访问 | 结构化接口 |
| 节点不可访问、样式复杂、视觉目标明显 | 截图+坐标 |
| 页面动画频繁 | 降低节奏或关闭等待，再决定是否改视觉闭环 |

## MJPEG 与端口转发

MJPEG 流端口 9100，适合低延迟连续画面分析。

```bash
iproxy 8100 8100 <UDID>   # 控制端口
iproxy 9100 9100 <UDID>   # MJPEG 端口
```

- 离散验证优先 `/screenshot`（恢复成本低）
- 仅动态追踪或高频分析才开 9100

## 扩展能力

| 能力 | 场景 | 注意 |
|------|------|------|
| 生物识别模拟 | Face ID/Touch ID 流程 | 依赖模拟器或上层驱动扩展 |
| `viewportScreenshot` | 排除状态栏/导航栏 | 常由上层驱动提供，非裸 WDA 标准 |

扩展能力失败 → 回退稳定主路径，不继续堆叠实验性接口。

## 完成检查

性能参数调整后确认是否引入误点、漏判或动作抢跑。不只看吞吐，同时检查成功率和恢复成本。
